# Security Policy

## Supported versions

Security fixes are expected to land on the default branch and the latest public release.

## Reporting a vulnerability

Please do not post exploit details or sensitive security material in a public issue.

Recommended path:

1. Use GitHub private vulnerability reporting if it is enabled for this repository.
2. If private reporting is not available, open a minimal public issue titled
   `Security contact requested` without technical details.
3. A maintainer can then move the conversation to a safer channel.

## What to include

Useful details:

- affected file, script, or workflow
- steps to reproduce
- impact
- suggested remediation, if known

## Scope

This repository contains automation, hooks, and documentation for a knowledge base
workflow. Security reports about accidental data exposure, path leaks, token leaks,
unsafe defaults, or remote-code-execution risks are especially valuable.
