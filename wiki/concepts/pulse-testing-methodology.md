# Pulse Testing Methodology

Подходы и workaround'ы для тестирования [[entities/pulse-messenger|Pulse Messenger]] на физических Android-устройствах, выработанные после столкновения с ограничениями [[concepts/flutter-adb-testing|ADB + Flutter]].

## Key Points

- **OTP через FCM push**: на обоих устройствах OTP приходит через push-уведомление и авто-заполняется — ручной ввод не нужен
- **Обход Flutter UI для авторизации**: OTP ставится вручную в Redis (`SET` ключ `"111111"`), затем верификация через прямой API-вызов
- **Прямое API-тестирование быстрее**: создание чатов и отправка сообщений через REST API, устройства — только для проверки отображения
- **`adb shell pm clear`** для сброса авторизации — быстрее, чем навигация к logout через UI
- **SMS_PROVIDER=mock** на сервере — OTP-коды логируются в stdout, но проще ставить напрямую в Redis

## Details

### Проблема Phantom Sessions

При тестировании обнаружена критическая проблема: Samsung работал с номером +79009876543, **которого не существовало в базе данных** — предыдущая тестовая сессия зарегистрировала номер некорректно, создав "призрачную" сессию без пользователя в БД. OnePlus был залогинен под "Стёпа" (реальный аккаунт разработчика), а не "TestUser1" — тестовых аккаунтов вообще не существовало.

**Урок**: всегда проверять состояние БД перед тестированием. Phantom sessions (UI показывает залогиненное состояние без записи в БД) могут привести к часам потерянного времени на отладку проблем синхронизации контактов.

### Workflow обхода авторизации

Когда [[concepts/flutter-adb-testing|ADB input не работает]] с Flutter TextField:

1. Очистить данные приложения: `adb shell pm clear <package>`
2. Установить OTP-код в Redis: `SET otp:<phone> "111111"`
3. Вызвать API верификации напрямую: `POST /api/auth/verify` с телефоном и OTP
4. Получить JWT-токен из ответа
5. Использовать токен для дальнейших API-вызовов (создание чатов, отправка сообщений)

### Используемые тестовые аккаунты

Вместо создания новых тестовых пользователей решено использовать существующие реальные аккаунты:
- "Стёпа" — +79001234567 (OnePlus)
- "Дима" — +79003456789 (Samsung)

### Статус тестирования

Этапы 1-3 (авторизация, OTP, профиль) завершены без багов. Этапы 4-13 (контакты, чаты, сообщения, группы, звонки, медиа, уведомления, мульти-устройство, настройки, надёжность, финальный отчёт) остаются незавершёнными.
