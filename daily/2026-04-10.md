# Daily Log — 2026-04-10

## [2026-04-10T14:30:00+03:00]

### Context

Работа над проектом AgentCorp (Office) — настройка Turborepo monorepo для SaaS-платформы с AI-агентами.

### Key Exchanges

- Обсуждение структуры monorepo: `apps/web` (Next.js 14), `apps/api` (Fastify), `packages/shared`
- Выбор между npm workspaces и pnpm — остановились на npm workspaces (совместимость с Turborepo из коробки)
- Настройка `turbo.json` с pipeline: build → lint → typecheck → test
- Проблема: TypeScript project references не работали с путями содержащими пробелы в Windows

### Decisions Made

- **Monorepo manager**: Turborepo + npm workspaces. Причина: нативная поддержка в Next.js, меньше конфигурации чем pnpm.
- **Shared packages**: `@agentcorp/ui` (shadcn компоненты), `@agentcorp/config` (ESLint/TS configs), `@agentcorp/types` (общие типы).
- **API framework**: Fastify вместо Express. Причина: встроенная валидация через JSON Schema, лучшая производительность, TypeScript-first.

### Lessons Learned

- Windows paths с пробелами ломают `tsconfig.json` project references — нужно использовать relative paths без пробелов или symlink.
- `turbo.json` pipeline dependency `^build` означает "сначала build зависимых пакетов" — без `^` будет параллельный build.
- Claude Code hooks в `~/.claude/settings.json` работают одинаково в CLI и VS Code extension — проверено по официальной документации.
- При настройке глобальных hooks путь с пробелами нужно оборачивать в escaped quotes: `\"path with spaces\"`.

### Action Items

- Добавить Prisma в `apps/api` с PostgreSQL
- Настроить Docker Compose для локальной разработки (postgres, redis, minio)

### Context

Интеграция LLM Wiki + Memory Compiler в глобальную базу знаний.

### Key Exchanges

- Изучены два инструмента: Karpathy's LLM Wiki (паттерн) и coleam00/claude-memory-compiler (реализация)
- Решено объединить оба подхода в единую wiki/ директорию
- Hooks настроены глобально — работают для всех проектов

### Decisions Made

- **Wiki location**: `E:\Project\memory claude\memory claude\` — отдельное git-репо, не внутри проектов
- **Unified wiki/**: ручной ingest (raw/ → wiki/sources/) и автокомпиляция (daily/ → wiki/concepts/) пишут в одну wiki/
- **Python стек**: uv + claude-agent-sdk для скриптов, не портируем на Node.js
- **Глобальные hooks**: SessionStart (инжект контекста), SessionEnd + PreCompact (захват transcript)

### Lessons Learned

- `claude-agent-sdk` требует Python >=3.12, `uv` автоматически выбирает подходящую версию из установленных
- Hook `SessionStart` может возвращать `additionalContext` через JSON stdout — до 10K символов
- `SessionEnd` и `PreCompact` получают `transcript_path` через stdin — можно извлечь полный разговор
- Recursion guard обязателен: flush.py → Agent SDK → Claude Code → hook → flush.py (бесконечный цикл без `CLAUDE_INVOKED_BY` check)
